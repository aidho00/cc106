02 Analogy

The Restaurant Analogy

UI = How the restaurant looks: decor, plates, menu design, lighting

UX = The whole experience: how quickly you're seated, how easy the menu is to read, how fast food arrives, how you feel when you leave

A beautiful restaurant (good UI) with terrible service (bad UX) still fails. You need BOTH.

02 One User Task

UI presents the choices; UX shapes the journey

UI decisions

  • Is the main button easy to notice?
  • Are labels and icons understandable?
  • Is the text readable?

UX decisions

  • Are the steps in a logical order?
  • Does the app explain errors clearly?
  • Can the user recover or go back?

A polished screen can still create a poor experience if the task is confusing, slow, or unforgiving.

02 Why It Matters

Good design helps users act with confidence

Understand

Users can quickly recognize the screen's purpose and available actions.

Complete

Clear steps, feedback, and error handling help users finish their task.

Trust

Consistency and readable information make the app feel dependable.

The goal is not decoration. The goal is to reduce confusion and support the user's purpose.

03 C.A.R.P.

4 Core Design Principles

C

Make important things stand out

A

Alignment

Everything connected visually

Repetition

Consistent patterns throughout

Proximity

Related items grouped together

03 Principle 1

Contrast

Make different things look DIFFERENT.

  • Big headings vs small body text
  • Bold vs light weight
  • Dark text on light background
  • Colored buttons vs plain text

Rule: If things are not the same, make them VERY different. No "kinda different."

03 Principle 2

Alignment

Every element should be visually connected to something else.

  • Left-align text (most common)
  • Align elements to a grid
  • Center-align sparingly (titles only)
  • Never mix alignments randomly

Rule: Nothing should look randomly placed.

✅ Consistent alignment

03 Principle 3

Repetition

Repeat visual elements throughout your app for consistency.

Repeat these:

  • Same

Why it works:

  • Creates visual unity
  • Users learn patterns fast
  • Looks professional
  • Reduces confusion
  • Easier to build (reuse components!)
03 Principle 4

Proximity

Related items should be grouped together. Unrelated items should have space between them.

  • Form label CLOSE to its input
  • Title CLOSE to its description
  • Group related buttons together
  • Separate different sections with space

Rule: Proximity implies relationship.

03 Worked Example

CARP principles work together on one form

Contrast: the Submit button is visually stronger than Cancel.

Alignment: labels, fields, and buttons share the same left edge.

Repetition: every field uses the same label style, border, and spacing.

Proximity: each label sits close to its field, while different sections have more space.

The principles are not separate decorations; together they make the form easier to scan and complete.

04 Palette Rule

The 60-30-10 Rule

60% — Dominant color (background, large areas)
30% — Secondary color (cards, sections, text)
10% — Accent color (buttons, links, highlights)

This creates visual balance. Too many colors = chaotic. Too few = boring. 3-4 colors max for most apps.

04 Tools

Where to Get Color Palettes

Free Tools

  • coolors.co — Generate random palettes
  • colorhunt.co — Curated palettes
  • materialpalettes.com — Material Design colors
  • Realtime Colors — Preview on a real UI

Quick Tips

  • Start with one color you like
  • Use tools to find complements
  • Test with real content, not lorem ipsum
  • Check contrast for readability
04 Use Color by Role

A palette becomes useful when each color has a job

Brand + action

Primary buttons, active navigation, important links.

Surface + text

Backgrounds, cards, borders, headings, and body copy.

Status

Success, warning, and error colors with matching icons or text.

Accessibility rule: never use color alone to communicate meaning. Pair it with a label, icon, pattern, or position.

Use for: body text, buttons, most UI

Serif

Traditional, elegant, authoritative

05 Type Scale

Font Size Hierarchy

Page Title (32-40px)
Section Heading (24-30px)
Subheading (18-22px)
Body text (14-16px)
Caption/label (12-14px)

Rule: Use 2-3 different sizes max per screen. More sizes = visual chaos.

05 Font Pairing

Font Pairing Tips

  • One display + one body font is enough for most apps
  • Max 2 fonts in your app. Never more than 3.
  • Google Fonts is free and works in FlutterFlow
  • Safe picks: Inter, Poppins, Nunito, Open Sans, Roboto
  • Avoid: Comic Sans, Papyrus, overly decorative fonts

FlutterFlow has Google Fonts built in. You can change fonts in 2 clicks.

06 Spacing

Whitespace is Your Friend

Empty space is NOT wasted space. It makes content breathable and readable.

  • Padding inside cards: 16-24px
  • Margin between sections: 24-40px
  • Line height: 1.4-1.6 for body text
  • Consistent spacing throughout

Cramped design = amateur look. Space = professional.

07 Design Beyond the Ideal Case

Every data screen needs more than its normal state

Loading

Show that the app is working and prevent repeated actions.

Empty

Explain why no content appears and suggest a useful next step.

Success

Confirm what changed and where the user can continue.

Error

Describe the problem in plain language and provide recovery.

These states are part of UX, not optional polish.

07 Avoid These

Top UI Mistakes

  • ❌ Too many colors (rainbow effect)
  • ❌ Tiny body text (below 16px on web)
  • ❌ No whitespace (everything cramped)
  • ❌ Inconsistent button styles
  • ❌ Poor contrast (light gray on white)
  • ❌ Random font sizes
  • ❌ No clear call-to-action
  • ❌ Walls of text
  • ❌ Unclear icons (what does this do?)
  • ❌ No loading states or feedback

08 Psychology

Gestalt Principles

How the human brain perceives visual information:

Similarity

Similar items look like they belong together. Same colored buttons = same type of action.

Closure

Brain fills in gaps. You don't need borders on all sides to see a "box."

Continuity

Eyes follow lines and curves. Horizontal lists, progress bars.

Figure/Ground

Users distinguish main content from background. Modals, overlays use this.

09 UX Rules

Key UX Principles

  • Don't make users think — If they're confused, your design failed
  • 3-step rule — Core actions within 3 clear steps
  • Feedback always — Every action should have a visible response
  • Forgiveness — Let users undo mistakes easily
  • Progressive disclosure — Show only what's needed now
  • Familiarity — Use patterns users already know
09 Accessibility

Design for Everyone

Not all users have perfect vision, motor skills, or tech literacy:

  • Contrast ratio: 4.5:1 minimum for text (use WebAIM contrast checker)
  • Touch targets: Minimum 44x44px for buttons (fat fingers!)
  • Font size: Never below 14px for body text
  • Color alone: Don't rely only on color to convey meaning (add icons/text)
  • Labels: Always label form fields, don't rely on placeholders
10 Your Checklist

UI/UX Checklist

Visual Design (UI)

  • ☐ Consistent color palette (3-4 colors)
  • ☐ Max 2 fonts
  • ☐ Clear visual hierarchy
  • ☐ Adequate whitespace
  • ☐ Consistent button/card styles
  • ☐ Good contrast (readable text)
  • ☐ Aligned elements (no random placement)

User Experience (UX)

  • ☐ Clear navigation (user always knows where they are)
  • ☐ Feedback on all actions
  • ☐ Error messages that help
  • ☐ Loading states shown
  • ☐ Core task in ≤ 3 clear steps
  • ☐ Empty states handled
  • ☐ Interactive controls ≥ 44px high
10 Recap

Key Takeaways

  • UI = looks, UX = feels. Both matter equally.
  • CARP: Contrast, Alignment, Repetition, Proximity
  • Colors: 3-4 max, 60-30-10 rule, colors have meaning
  • Typography: 2 fonts max, clear size hierarchy
  • Whitespace: Your best friend. Use it generously.
  • Visual hierarchy: Guide the eye from most to least important
  • Patterns: Use what users already know (Material Design)
